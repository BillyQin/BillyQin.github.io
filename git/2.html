    # git分支管理规范

    # 1. 主分支master

    用于发布正式版本以及打tag

    # 2. 日常开发分支 develop

    用与日常开发

    # 3. 其他临时分支

    • feature 功能分支 功能分支，从develop创建，开发完成后合入develop分支
    • release 预发布分支 发布正式版本前，使用预发布分支测试。测试完成后合入master，并打tag
    • fixbug 修补bug分支 从master分支引出，修改完后合入develop和master
